SQL Quick Notes

DDL – Data Definition Language

DDL is used to define the structure of a database. It includes commands like CREATE, ALTER, DROP, and TRUNCATE to define, modify, and remove structures such as tables, views, and indexes.

DQL – Data Query Language

DQL is used to query the database and retrieve data. The main command is SELECT.

DML – Data Manipulation Language

DML is used for managing data within schema objects. The main commands are INSERT, UPDATE, and DELETE.

DCL – Data Control Language

DCL includes commands like GRANT and REVOKE, which are used to control access and permissions to data and database objects.

TCL – Transaction Control Language

TCL is used to manage transactions in a database. Common commands include COMMIT, ROLLBACK, and SAVEPOINT.

Important Note: DDL, DML, DQL, DCL, and TCL are the five types of SQL commands, each serving different purposes in the database management system.

Data Lake vs Data Warehouse

Data Lake: A storage repository that can hold large amounts of raw, unstructured, and structured data. It is highly scalable and cost-effective, often used for big data processing and analytics.

Data Warehouse: A structured and optimized database for reporting and analysis. Data warehouses store data from different sources in a more refined, structured format, often used for business intelligence applications.

SQL vs MySQL

SQL = Language.
MySQL = Software (RDBMS).

Table and Fields

Table = Rows + Columns.
Fields = Columns.
Rows = Records.

Primary Key

Uniquely identifies each row.

Foreign Key

Refers to Primary Key in another table. Child and Parent tables.

Joins

  • INNER JOIN: Matching records in both tables.
  • LEFT JOIN: All records from left + matching from right.
  • RIGHT JOIN: All records from right + matching from left.
  • FULL JOIN: All records from both tables.
  • SELF JOIN: Table joined with itself.
  • CROSS JOIN: Combines every row of both tables.

Query

A request for data.

Subquery

A query inside another query.

SELECT

Used to select data from a database.

Clauses with SELECT

  • WHERE: Filter records.
  • ORDER BY: Sort records (ASC/DESC).
  • GROUP BY: Group same data.
  • HAVING: Filter grouped records.

Entity

Real-world object.

Relationships

Links between entities.

Cardinality

One-to-One, One-to-Many, Many-to-Many.

Alias

Temporary name for a table or column.

Normalization

Remove redundancy.

Types of Normalization

  • 1NF: Atomic fields, primary key, no repeating groups.
  • 2NF: 1NF + remove partial dependency.
  • 3NF: 2NF + remove transitive dependency.

Delete vs Truncate vs Drop

  • DELETE: Remove selected rows.
  • TRUNCATE: Remove all rows.
  • DROP: Remove whole object (table, database).

Aggregate Functions

  • AVG()
  • COUNT()
  • MIN()
  • MAX()
  • SUM()
  • FIRST()
  • LAST()

Wildcards

  • % : any number of characters.
  • _ : single character.
  • [] : match any character inside.
  • - : range inside [].

View

Save a query as a fake table.

CREATE VIEW top_students AS
SELECT name FROM students WHERE marks > 80;
SELECT * FROM top_students;

Index

Search faster by creating an index.

CREATE INDEX idx_student_name ON students(name);

Stored Procedure

Save steps and run them later.

CREATE PROCEDURE show_students()
BEGIN
  SELECT * FROM students;
END;
CALL show_students();

OLTP vs OLAP

  • OLTP: Daily work (Insert, Update). Example: ATM, Online Shopping.
  • OLAP: Analyze big data. Example: Business Reports, Sales Analysis.
